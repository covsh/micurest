About

This is an example of implementing REST services for an ESP8266, or more specifically – NodeMcu SOM. The intent of this example is to demonstrate μcuREST capabilities and show technics for implementing various REST resources with μcuREST: plain variables, complex objects, custom-stringified variables (such as IP and MAC addresses), web pages and binary data.
As a side result, this example facilitates configuration interface for the SOM.

This example exposes a set of URI’s for communicating between SOM and the browser.

URI Methods Description
/status GET Returns bootstrap html page with embedded SOM status data
/restart PUT Performs soft reset of MCU
/sysinfo GET[0] Returns JSON object with SOM status data
/sysinfo/rtdata GET Returns JSON object with SOM real time data
/sysinfo/interfaces GET Returns JSON object with network interfaces data
/setup GET Returns bootstrap html page with SOM configuration data
/config All [0], [1], [2] Returns/accepts JSON object with current SOM configuration data
/config/current All Returns/accepts JSON object with current SOM configuration data
/config/default All Returns/accepts JSON object with default SOM configuration data
/config/ap All[0] Returns/accepts JSON object with current WiFi Access Point configuration
/config/ap/current All Returns/accepts JSON object with current WiFi Access Point configuration
/config/ap/default All Returns/accepts JSON object with default WiFi Access Point configuration
/config/ap/dhcps All Returns/accepts JSON object with DHCP server configuration
/config/sta All [0] Returns/accepts JSON object with current WiFi Station configuration
/config/sta/current All Returns/accepts JSON object with current WiFi Station configuration
/config/sta/default All Returns/accepts JSON object with default WiFi Station configuration
/config/sntp All Returns/accepts JSON object with SNTP configuration
/config/gpio All[0] Returns/accepts JSON object with GPIO configuration (all pins)
/config/gpio/* All Returns/accepts JSON object with GPIO configuration (all pins)
/config/gpio/<N> All Returns/accepts JSON object with GPIO configuration (pin <N>)
/config/port/0 All Returns/accepts JSON object with port configuration bit mask
/config/pad/<N> All Returns/accepts JSON object with GPIO configuration for MCU pad<N>
/meta GET[0] Returns JSON object with meta data about GPIO
/gpio GET Returns bootstrap html page with GPIO configuration data
/meta/* GET Returns JSON object with meta data about GPIO
/meta/mode GET Returns JSON array with valid mode values
/port/0 All Returns GPIO inputs as a bit mask
/demo GET Returns bootstrap html page with an enum - list of pin modes
/demo/index.html GET Returns bootstrap html page with an enum - list of pin modes
/demo/logical All Returns/accepts content of variable[3] bool logical
/demo/natural All Returns/accepts content of variable unsigned natural
/demo/text All Returns/accepts content of variable char text[32]
/demo/mode All Returns/accepts content of variable enum mode_t mode
/demo/blob All Returns/accepts binary object – content of array byte blob[1400]

For more information about μcuREST please visit these links:

  • μcuREST - An application server for constrained platforms
  • μcuREST Tutorial

Source code for this example can be found on github.


[0] Accept header must include application/json
[1] All supported method: GET/PUT/POST
[2] POST/PUT data must not include sections a and in
[3] All variables are allocated on PLC’s RAM
